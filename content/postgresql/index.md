This **PostgreSQL tutorial** helps you quickly understand PostgreSQL. You’ll master PostgreSQL fast through many practical examples and apply the knowledge to application development using PostgreSQL.

If you want to:

- Learn PostgreSQL fast.
- Develop applications using PostgreSQL as the back-end database.
- Migrate from other database management systems such as MySQL, Oracle, and Microsoft SQL Server.

You’ve come to the right place. This website provides you with everything you need to know to get started with PostgreSQL quickly and effectively.

The PostgreSQL tutorial demonstrates many unique features of PostgreSQL that position it as the most advanced open-source database management system.

## Quick start

[Getting Started with PostgreSQL](/postgresql/postgresql-getting-started)

This section helps you get started with PostgreSQL by showing you how to install PostgreSQL on Windows, Linux, and macOS. You also learn how to connect to PostgreSQL using the psql tool as well as how to load a sample database into the PostgreSQL for practicing.

## Section 1. Querying Data

- **[Select](/postgresql/postgresql-tutorial/postgresql-select)** – Show you how to query data from a single table.
- **[Column Aliases](/postgresql/postgresql-tutorial/postgresql-column-alias)** – Learn how to assign temporary names to columns or expressions within a query.
- **[Order By](/postgresql/postgresql-tutorial/postgresql-order-by)** – Guide you on how to sort the result set returned by a query.
- **[Select Distinct](/postgresql/postgresql-tutorial/postgresql-select-distinct)** – Show you how to remove duplicate rows from the result set.

## Section 2. Filtering Data

- **[Where](/postgresql/postgresql-tutorial/postgresql-where)** – Filter rows based on a specified condition.
- **[AND operator](/postgresql/postgresql-tutorial/postgresql-and)** – Combine two boolean expressions and return true if both expressions evaluate to true.
- **[OR operator](/postgresql/postgresql-tutorial/postgresql-or)** – Combine two boolean expressions and return false if either expression evaluates to false.
- **[Limit](/postgresql/postgresql-tutorial/postgresql-limit)** – Retrieve a subset of rows generated by a query.
- **[Fetch](/postgresql/postgresql-tutorial/postgresql-fetch)** – Limit the number of rows returned by a query.
- **[In](/postgresql/postgresql-tutorial/postgresql-in)** – Select data that matches any value in a list of values.
- **[Between](/postgresql/postgresql-tutorial/postgresql-between)** – Select data that is a range of values.
- **[Like](/postgresql/postgresql-tutorial/postgresql-like)** – Filter data based on pattern matching.
- **[Is Null](/postgresql/postgresql-tutorial/postgresql-is-null)** – Check if a value is null or not.

## Section 3. Joining Multiple Tables

- **[Joins](/postgresql/postgresql-tutorial/postgresql-joins)** – Show you a brief overview of joins in PostgreSQL.
- **[Table Aliases](/postgresql/postgresql-tutorial/postgresql-alias)** – Describes how to use table aliases in the query.
- **[Inner Join](/postgresql/postgresql-tutorial/postgresql-inner-join)** – Select rows from one table that have corresponding rows in other tables.
- **[Left Join](/postgresql/postgresql-tutorial/postgresql-left-join)** – Select rows from one table that may or may not have corresponding rows in other tables.
- **[Self-join](/postgresql/postgresql-tutorial/postgresql-self-join)** – Join a table to itself by comparing a table to itself.
- **[Full Outer Join](/postgresql/postgresql-tutorial/postgresql-full-outer-join)** – Use the full join to find rows in a table that do not have matching rows in another table.
- **[Cross Join](/postgresql/postgresql-tutorial/postgresql-cross-join)** – Produce a Cartesian product of the rows in two or more tables.
- **[Natural Join](/postgresql/postgresql-tutorial/postgresql-natural-join)** – Join two or more tables using implicit join conditions based on the common column names in the joined tables.

## Section 4. Grouping Data

- **[Group By](/postgresql/postgresql-tutorial/postgresql-group-by)** – divide rows of a result set into groups and optionally apply an aggregate function to each group.
- **[Having](/postgresql/postgresql-tutorial/postgresql-having)** – apply conditions to groups, which allow you to filter groups.

## Section 5. Set Operations

- **[Union](/postgresql/postgresql-tutorial/postgresql-union)** – combine result sets of multiple queries into a single result set.
- **[Intersect](/postgresql/postgresql-tutorial/postgresql-intersect)** – combine the result sets of two or more queries and return a single result set containing rows that appear in both result sets.
- **[Except](/postgresql/postgresql-tutorial/postgresql-except)** – return the rows from the first query that do not appear in the output of the second query.

## Section 6. Grouping Sets, Cubes, and Rollups

- **[Grouping Sets](/postgresql/postgresql-tutorial/postgresql-grouping-sets)** – generate multiple grouping sets in reporting.
- **[Cube](/postgresql/postgresql-tutorial/postgresql-cube)** – define multiple grouping sets that include all possible combinations of dimensions.
- **[Rollup](/postgresql/postgresql-tutorial/postgresql-rollup)** – generate reports that contain totals and subtotals.

## Section 7. Subquery

- **[Subquery](/postgresql/postgresql-tutorial/postgresql-subquery)** – write a query nested inside another query.
- **[Correlated Subquery](/postgresql/postgresql-tutorial/postgresql-correlated-subquery)** – show you how to use a correlated subquery.
- **[ANY](/postgresql/postgresql-tutorial/postgresql-any)** – retrieve data by comparing a value with a set of values returned by a subquery.
- **[ALL](/postgresql/postgresql-tutorial/postgresql-all)** – query data by comparing a value with a list of values returned by a subquery.
- **[EXISTS](/postgresql/postgresql-tutorial/postgresql-exists)** – check for the existence of rows returned by a subquery.
