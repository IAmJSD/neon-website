This**PostgreSQL tutorial** helps you quickly understand PostgreSQL. You'll master PostgreSQL fast through many practical examples and apply the knowledge to application development using PostgreSQL.

If you want to:

- Learn PostgreSQL fast.
-
- Develop applications using PostgreSQL as the back-end database.
-
- Migrate from other database management systems such as MySQL, Oracle, and Microsoft SQL Server.

You've come to the right place. This website provides you with everything you need to know to get started with PostgreSQL quickly and effectively.

The PostgreSQL tutorial demonstrates many unique features of PostgreSQL that position it as the most advanced open-source database management system.

## Quick start

## Basic PostgreSQL Tutorial

First, you'll learn how to query data from a single table using basic data techniques, which include selecting data, sorting result sets, and filtering rows.

Next, you'll delve into advanced queries, which include joining multiple tables, using set operations, and constructing the subquery.

Finally, you'll learn how to manage database tables, including creating a new table or modifying the structure of an existing table.

### Section 1. Querying Data

- [Select](/postgresql/postgresql-select "PostgreSQL SELECT") - show you how to query data from a single table.
-
- [Column aliases](/postgresql/postgresql-column-alias) - learn how to assign temporary names to columns or expressions within a query.
-
- [Order By](/postgresql/postgresql-order-by "PostgreSQL ORDER BY") - guide you on how to sort the result set returned by a query.
-
- [Select Distinct](/postgresql/postgresql-select-distinct "PostgreSQL SELECT DISTINCT")- show you how to remove duplicate rows from the result set.

### Section 2. Filtering Data

- [Where](/postgresql/postgresql-where "PostgreSQL WHERE") - filter rows based on a specified condition.
-
- [AND operator](/postgresql/postgresql-and) - combine two boolean expressions and return true if both expressions evaluate to true.
-
- [OR operator](/postgresql/postgresql-or) - combine two boolean expressions and return false if either expression evaluates to false.
-
- [Limit](/postgresql/postgresql-limit) - retrieve a subset of rows generated by a query.
-
- [Fetch](/postgresql/postgresql-fetch)- limit the number of rows returned by a query.
-
- [In](/postgresql/postgresql-in "PostgreSQL IN") - select data that matches any value in a list of values.
-
- [Between](/postgresql/postgresql-between "PostgreSQL BETWEEN") - select data that is a range of values.
-
- [Like](/postgresql/postgresql-like "PostgreSQL LIKE") - filter data based on pattern matching.
-
- [Is Null](/postgresql/postgresql-is-null) - check if a value is null or not.

### Section 3. Joining Multiple Tables

- [Joins](/postgresql/postgresql-joins) - show you a brief overview of joins in PostgreSQL.
-
- [Table aliases](/postgresql/postgresql-alias) - describes how to use table aliases in the query.
-
- [Inner Join](/postgresql/postgresql-inner-join "PostgreSQL INNER JOIN")- select rows from one table that have the corresponding rows in other tables.
-
- [Left Join](/postgresql/postgresql-left-join "PostgreSQL LEFT JOIN") - select rows from one table that may or may not have the corresponding rows in other tables.
-
- [Self-join](/postgresql/postgresql-self-join) - join a table to itself by comparing a table to itself.
-
- [Full Outer Join](/postgresql/postgresql-full-outer-join) - use the full join to find a row in a table that does not have a matching row in another table.
-
- [Cross Join](/postgresql/postgresql-cross-join) - produce a Cartesian product of the rows in two or more tables.
-
- [Natural Join](/postgresql/postgresql-natural-join) - join two or more tables using implicit join conditions based on the common column names in the joined tables.

### Section 4. Grouping Data

- [Group By](/postgresql/postgresql-group-by "PostgreSQL GROUP BY") - divide rows of a result set into groups and optionally apply an aggregate function to each group.
-
- [Having](/postgresql/postgresql-having "PostgreSQL HAVING") - apply conditions to groups, which allow you to filter groups.

### Section 5. Set Operations

- [Union](/postgresql/postgresql-union "PostgreSQL UNION") - combine result sets of multiple queries into a single result set.
-
- [Intersect](/postgresql/postgresql-intersect) - combine the result sets of two or more queries and return a single result set containing rows that appear in both result sets.
-
- [Except](/postgresql/postgresql-except) - return the rows from the first query that do not appear in the output of the second query.

### Section 6. Grouping sets, Cubes, and Rollups

- [Grouping Sets](/postgresql/postgresql-grouping-sets) - generate multiple grouping sets in reporting.
-
- [Cube](/postgresql/postgresql-cube) - define multiple grouping sets that include all possible combinations of dimensions.
-
- [Rollup](/postgresql/postgresql-rollup) - generate reports that contain totals and subtotals.

### Section 7. Subquery

- [Subquery](/postgresql/postgresql-subquery "PostgreSQL Subquery") - write a query nested inside another query.
-
- [Correlated Subquery](/postgresql/postgresql-correlated-subquery) - show you how to use a correlated subquery to perform a query that depends on the values of the current row being processed.
-
- [ANY](/postgresql/postgresql-any) - retrieve data by comparing a value with a set of values returned by a subquery.
-
- [ALL](/postgresql/postgresql-all) - query data by comparing a value with a list of values returned by a subquery.
-
- [EXISTS](/postgresql/postgresql-exists) - check for the existence of rows returned by a subquery.

### Section 8. Common Table Expressions

- [Update](/postgresql/postgresql-update "PostgreSQL UPDATE") - update existing data in a table.
-
- [Update join](/postgresql/postgresql-update-join) - update values in a table based on values in another table.
-
- [Delete](/postgresql/postgresql-delete "PostgreSQL DELETE") - delete data in a table.

- [Upsert](/postgresql/postgresql-upsert)- insert or update data if the new row already exists in the table.
